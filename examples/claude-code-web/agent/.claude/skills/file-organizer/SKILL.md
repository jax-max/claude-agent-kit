# File Organizer（文件组织）

专业的文件组织技能，帮助整理项目文件结构、创建合理的目录层次、移动和重组文件、清理无用文件。

## 何时使用此技能

在以下场景使用此技能：
- 整理混乱的项目结构
- 重构目录组织方式
- 创建新的目录结构
- 移动文件到合适的位置
- 清理无用文件和临时文件
- 统一文件命名规范
- 组织配置文件
- 分离不同类型的文件（源代码、测试、文档等）

## 如何使用

### 1. 分析当前结构

首先了解项目现状：
1. 使用 `LS` 或 `Glob` 查看当前目录结构
2. 识别文件类型和用途
3. 找出结构问题（如文件散乱、命名不一致等）
4. 了解项目的技术栈和类型

### 2. 设计新的结构

根据项目类型设计合理的目录结构：

**通用项目结构：**
```
project/
├── src/              # 源代码
│   ├── components/   # 组件
│   ├── utils/        # 工具函数
│   ├── types/        # 类型定义
│   └── ...
├── tests/            # 测试文件
├── docs/             # 文档
├── config/           # 配置文件
├── scripts/          # 脚本文件
└── ...
```

**Node.js/TypeScript 项目：**
```
project/
├── src/
├── dist/             # 编译输出
├── tests/
├── docs/
├── config/
├── scripts/
├── package.json
└── tsconfig.json
```

### 3. 执行重组

1. **创建新目录**：使用 `Write` 创建必要的目录结构
2. **移动文件**：使用 `Bash` 命令移动文件到新位置
3. **更新引用**：使用 `Grep` 查找并更新文件中的导入路径
4. **更新配置**：修改配置文件中的路径引用

### 4. 清理无用文件

识别并清理：
- 临时文件（`.tmp`, `.log`, `.cache`）
- 备份文件（`.bak`, `~`, `.old`）
- 构建产物（`node_modules`, `dist`, `.next`）
- 重复文件
- 未使用的文件

## 常见任务

### 任务1：按类型组织文件

**场景**：项目中所有文件都在根目录，需要按类型分类

**步骤**：
1. 识别文件类型（组件、工具、配置等）
2. 创建对应的目录
3. 移动文件到相应目录
4. 更新所有导入路径

### 任务2：分离源代码和测试

**场景**：测试文件混在源代码中

**步骤**：
1. 创建 `tests/` 或 `__tests__/` 目录
2. 识别测试文件（`*.test.ts`, `*.spec.ts`）
3. 移动测试文件
4. 更新测试配置中的路径

### 任务3：统一命名规范

**场景**：文件名命名不一致（驼峰、短横线、下划线混用）

**步骤**：
1. 确定项目命名规范（如 kebab-case）
2. 识别不符合规范的文件
3. 重命名文件
4. 更新所有引用

### 任务4：创建标准项目结构

**场景**：新项目或需要标准化现有项目

**步骤**：
1. 根据项目类型创建标准目录结构
2. 创建必要的配置文件
3. 添加 README.md 说明结构
4. 移动现有文件到新结构

## 目录结构建议

### 小型项目
```
project/
├── src/
├── tests/
├── docs/
└── README.md
```

### 中型项目
```
project/
├── src/
│   ├── components/
│   ├── utils/
│   ├── types/
│   └── index.ts
├── tests/
├── docs/
├── config/
└── README.md
```

### 大型项目
```
project/
├── src/
│   ├── features/      # 按功能模块组织
│   ├── shared/        # 共享代码
│   ├── core/          # 核心功能
│   └── ...
├── tests/
├── docs/
├── config/
├── scripts/
└── README.md
```

## 文件命名规范

### 推荐规范
- **源代码文件**：kebab-case（`user-service.ts`）
- **组件文件**：PascalCase（`UserProfile.tsx`）
- **配置文件**：kebab-case（`tsconfig.json`, `webpack.config.js`）
- **测试文件**：与源文件同名 + `.test`（`user-service.test.ts`）
- **文档文件**：UPPERCASE（`README.md`, `CHANGELOG.md`）

### 避免的命名
- 空格和特殊字符
- 中文文件名（除非必要）
- 过长的文件名
- 无意义的缩写

## 执行步骤示例

### 示例：整理 TypeScript 项目

```markdown
## 文件组织计划

### 当前状态
- 所有文件在根目录
- 测试文件混在源代码中
- 配置文件散乱

### 目标结构
```
project/
├── src/
│   ├── components/
│   ├── utils/
│   ├── types/
│   └── index.ts
├── tests/
├── config/
└── ...
```

### 执行步骤
1. ✅ 创建 src/ 目录
2. ✅ 创建 tests/ 目录
3. ✅ 创建 config/ 目录
4. ✅ 移动 .ts 文件到 src/
5. ✅ 移动 .test.ts 文件到 tests/
6. ✅ 移动配置文件到 config/
7. ✅ 更新 tsconfig.json 路径
8. ✅ 更新所有 import 语句
```

## 注意事项

- **备份重要文件**：在执行大规模重组前，建议先备份
- **更新引用**：移动文件后必须更新所有导入路径
- **保持一致性**：整个项目使用统一的命名和组织规范
- **考虑工具链**：确保新的结构符合构建工具和框架的要求
- **渐进式重构**：大型项目可以分步骤重组，避免一次性大改动
- **文档更新**：重组后更新 README 和文档中的路径说明
